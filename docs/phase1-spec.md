# ClimbPost — PHASE 1 기능 명세서

> **범위:** 전체 파이프라인 + 필수 기능
> **기간:** 2026.03.15 ~ 2026.04.04 (3주)
> **플랫폼:** iOS (iPhone)

---

## 전체 파이프라인 요약

```
[iOS 앱]                    [분석 서버 (GPU)]              [iOS 앱]
갤러리 스캔 → 업로드 ────→ 클리핑 → 분류 → 태깅 → 편집 ────→ 결과 표출 → 인스타 공유
                                                          ← 푸시 알림 ←
```

---

## 모듈 1: 인증 (Auth)

### 1-1. 소셜 로그인

| 항목 | 내용 |
|------|------|
| 기능 | Apple Sign In, Google Sign In |
| 저장 데이터 | user_id, email, provider, created_at |
| 프로필 | 없음 (PHASE 1에서는 미구현) |
| 세션 관리 | JWT 토큰 기반 |

**수락 기준:**
- 앱 첫 실행 시 로그인 화면 표시
- Apple / Google 로그인 성공 시 메인 화면 진입
- 토큰 만료 전까지 자동 로그인 유지

## 모듈 2: 갤러리 스캔 & 필터링 (Gallery)

### 2-1. 갤러리 접근 권한

| 항목 | 내용 |
|------|------|
| 권한 | Photos 프레임워크 접근 (PHPhotoLibrary) |
| 범위 | 영상만 (mediaType == .video) |
| 시간 범위 | 당일 촬영분 (우선 당일 목표) |

### 2-2. 클라이밍 영상 자동 필터링

| 항목 | 내용 |
|------|------|
| 방식 | PHAsset의 `location` + `creationDate` 활용 |
| 장소 판별 | 암장 GPS DB와 좌표 매칭 (반경 기준) |
| 데이터 소스 | `data/gyms.json` (개발자 직접 관리) |

**필터링 로직:**
```
1. PHAsset에서 당일 촬영 영상 전체 조회
2. 각 영상의 location(CLLocation)을 암장 GPS DB와 비교
3. 반경 N미터 이내 → 클라이밍 영상으로 판별
4. 매칭된 영상 목록을 사용자에게 표시
```

**gyms.json 구조 예시:**
```json
{
  "gyms": [
    {
      "id": "gym_001",
      "name": "더클라임 강남",
      "latitude": 37.4967,
      "longitude": 127.0276,
      "color_map": {
        "노랑": "V0-V1",
        "초록": "V2-V3",
        "파랑": "V4-V5",
        "빨강": "V6-V7",
        "검정": "V8+"
      }
    }
  ]
}
```

### 2-3. 영상 업로드

| 항목 | 내용 |
|------|------|
| 대상 | 필터링된 클라이밍 영상 (약 20개, 1~10분) |
| 전송 | 분석 서버로 업로드 |
| UX | 업로드 진행률 표시 → 업로드 완료 후 앱 종료 가능 |
| 용량 고려 | 4K 기준 1분 ≈ 350MB, 최대 수십 GB 가능 |

**수락 기준:**
- 앱 실행 시 당일 클라이밍 영상 자동 감지
- 감지된 영상 목록 표시 및 확인
- 업로드 시작 후 앱을 닫아도 백그라운드 업로드 지속
- 업로드 완료 시 서버에서 분석 자동 시작

## 모듈 3: 영상 분석 (Analyzer)

### 3-1. 클라이밍 구간 추출 (Clipper)

| 항목 | 내용 |
|------|------|
| 입력 | raw video (1~10분) |
| 출력 | clip video 목록 (벽에 붙어 있는 구간만) |
| 방식 | AI 기반 자동 감지 (모델 선택은 개발 단계에서 결정) |
| 조건 | 고정 카메라 (삼각대), 세로 영상 |

**클리핑 기준:**
- 사람이 벽에 접근하여 등반 시작하는 시점 = 클립 시작
- 벽에서 떨어지거나 탑아웃하는 시점 = 클립 종료
- 클립 앞뒤 여유 시간(buffer) 포함

### 3-2. 성공/실패 분류 (Classifier)

| 항목 | 내용 |
|------|------|
| 입력 | clip video |
| 출력 | success / fail 라벨 |

**판별 기준:**
- **성공 조건 (OR):**
  - 최상단 홀드 도달 후 일정 시간(N초) 유지
  - 탑아웃 (꼭대기를 넘어감)
  - 등반 완료 후 스스로 안전하게 하강
- **실패 조건:**
  - 등반 중 추락 (벽에서 떨어짐)

### 3-3. 난이도 태깅 (Detector)

| 항목 | 내용 |
|------|------|
| 입력 | clip video + 암장 color_map |
| 출력 | 난이도 라벨 (V스케일) |
| 방식 | 테이프 색상 인식 → 암장별 매핑 테이블 참조 |

**처리 흐름:**
```
1. 클립 영상에서 등반자가 사용하는 홀드 영역의 테이프 감지
2. 테이프의 주요 색상 추출
3. 해당 암장의 color_map에서 색상 → 난이도 매핑
4. 메타데이터에 난이도 기록
```

### 3-4. 사용자 식별 (Identifier)

| 항목 | 내용 |
|------|------|
| 입력 | 당일 전체 clip video |
| 출력 | 각 클립의 등반자가 "나"인지 여부 |
| 방식 | 실시간 적응형 학습 — 출현 빈도 기반 |

**식별 로직:**
```
1. 삼각대 촬영 → 촬영자는 프레임에 미등장
2. 모든 클립에서 등반자 외형(체형, 의상 등) 특징 추출
3. 클러스터링 → 가장 빈도 높은 클러스터 = "나"
4. 각 클립에 is_me: true/false 태깅
```

### 3-5. 영상 편집 (Editor)

| 항목 | 내용 |
|------|------|
| 입력 | 선택된 clip videos |
| 출력 | 인스타그램 캐러셀용 편집 영상 |
| 비율 | 3:4 (세로) |
| 길이 | 클립당 최대 60초 |
| 형태 | 캐러셀 — 여러 클립을 한 게시물로 |

## 모듈 4: 결과 표출 & 공유 (Result)

### 4-1. 푸시 알림

| 항목 | 내용 |
|------|------|
| 트리거 | 분석 서버에서 처리 완료 시 |
| 내용 | "분석 완료! 결과를 확인해보세요" |
| 동작 | 알림 탭 → 앱 결과 화면으로 이동 |

### 4-2. 클립 목록 & 필터

| 항목 | 내용 |
|------|------|
| 표시 정보 | 썸네일, 난이도, 성공/실패, 등반자(나/타인) |
| 필터 | 난이도별, 성공/실패, 나의 클립만 |
| 재생 | 클립 선택 시 미리보기 재생 |

### 4-3. 캐러셀 구성 & 인스타 공유

| 항목 | 내용 |
|------|------|
| 클립 선택 | 유저가 올릴 클립 선택 + 순서 결정 |
| 미리보기 | 편집된 캐러셀 최종 확인 |
| 공유 방식 | iOS Share Sheet → 인스타그램 앱으로 전달 |
| 유저 액션 | 인스타 앱에서 "게시" 버튼만 누르면 완료 |
