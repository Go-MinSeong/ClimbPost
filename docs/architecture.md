- AI 모델 추론 (GPU 가속)
- 처리 결과 + 편집된 클립을 Backend API로 전달
- 처리 완료 시 푸시 알림 트리거

- HTTPS 필수 (Let's Encrypt 또는 self-signed)

## 6. 분석 파이프라인 상세

```
raw_video (입력)
     │
     ▼
┌─────────────────────────────────────────────────┐
│  Stage 1: Clipper (구간 추출)                     │
│                                                 │
│  AI 모델로 "사람이 벽에 붙어 있는 구간" 감지         │
│  - 포즈 추정 모델 활용 가능                        │
│  - 사람의 수직 이동 패턴 분석                      │
│                                                 │
│  입력: raw_video (1~10분)                        │
│  출력: clip_videos[] (각 등반 시도별 분리)          │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│  Stage 2: Classifier (성공/실패)                  │
│                                                 │
│  클립의 마지막 부분 분석                           │
│  - 꼭대기 도달 + 유지 → success                   │
│  - 탑아웃 → success                              │
│  - 추락 → fail                                   │
│                                                 │
│  입력: clip_video                                │
│  출력: result (success / fail)                   │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│  Stage 3: Detector (난이도 태깅)                  │
│                                                 │
│  테이프 색상 인식 → 암장 매핑 테이블 참조              │
│  - 객체 감지로 테이프 영역 추출                     │
│  - 색상 분석 (HSV 또는 딥러닝)                     │
│  - color_map에서 난이도 변환                      │
│                                                 │
│  입력: clip_video + gym.color_map               │
│  출력: difficulty (V스케일), tape_color           │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│  Stage 4: Identifier (사용자 식별)                │
│                                                 │
│  당일 전체 클립에서 등반자 클러스터링                │
│  - 사람 재식별(Re-ID) 또는 외형 특징 기반           │
│  - 최다 출현 클러스터 = "나"                       │
│                                                 │
│  입력: 당일 전체 clips[]                          │
│  출력: 각 clip.is_me (boolean)                   │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│  Stage 5: Editor (영상 편집)                      │
│                                                 │
│  인스타그램 캐러셀 규격에 맞춰 편집                  │
│  - 비율: 3:4 (세로)                              │
│  - 클립당 최대 60초                               │
│  - 불필요 구간 트리밍                              │
│                                                 │
│  입력: clip_videos[]                             │
│  출력: edited_videos[] (업로드 준비 완료)           │
└─────────────────────────────────────────────────┘
```

## 9. PHASE별 아키텍처 진화 방향

| 항목 | PHASE 1 | PHASE 2+ |
|------|---------|----------|
| 서버 | 집 GPU 1대 (API + 분석 동일 서버) | 클라우드 분리 (API↔분석) |
| DB | SQLite 또는 PostgreSQL (로컬) | 클라우드 DB (RDS 등) |
| 스토리지 | 로컬 디스크 | S3 / Cloud Storage |
| 큐 | 인메모리 또는 Redis | RabbitMQ / SQS |
| 인증 | JWT (자체) | OAuth 2.0 고도화 |
| 모니터링 | 로그 파일 | Grafana / Sentry |
